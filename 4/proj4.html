Part 2: Recovering Homographies

I computed homographies between image pairs using manually defined point correspondences and solved the linear system using least-squares to obtain the transformation matrix. The system of equations was of the form: [-x2, -y2, -1, 0, 0, 0, x1 * x2, x1 * y2, x1] and [0, 0, 0, -x2, -y2, -1, y1 * x2, y1 * y2, y1] allowing me to solve for the unknowns of the 3x3 homography matrix (with one entry being 1). I selected many correspondence points to order to have an over-determined system of equations so that my solution is more robust and accurate. This system allows us to recover the homography matrix which allows warping between images.

Part 3: Warping the Images

Using the recovered homographies, I warped images to align with the reference image. I implemented forward and inverse warping techniques, ensuring proper interpolation using scipy's map_coordinates to create the stitched mosaics. As a sanity check, I took pictures of posters that were at an angle in Haas Pavilion and then used the Warping to see if I can warp the poster onto a flat plane. This was done by setting the correspondences of the first image to be the corners of the poster and the correspondences of the flat plane to be rectangular coordinates of the form: [0, 0; 0, d; d, d; d, 0].

Part 5: Automatic Image Stitching System

The goal of this project is to create a system for automatically stitching images into a mosaic. A secondary goal is to learn how to read and implement a research paper. The project consists of the following steps:

Steps 1-3: Feature Detection and Matching

For steps 1-3, we followed the paper “Multi-Image Matching using Multi-Scale Oriented Patches” by Brown et al. with several simplifications. The implementation process is outlined below:

Step 1: Detecting Corner Features

We started with the Harris Interest Point Detector (Section 2). We used a single scale and did not implement sub-pixel accuracy. Below is an example of the Harris corners overlaid on the image.

Harris Corners Overlay

Step 2: Adaptive Non-Maximal Suppression

Next, we implemented Adaptive Non-Maximal Suppression (Section 3) to select a refined set of corner points. This step ensures that the selected corners are well-distributed and prominent. Below is a figure showing the chosen corners overlaid on the image.

Adaptive Non-Maximal Suppression Overlay

Step 3: Feature Descriptor Extraction

We extracted feature descriptors by capturing axis-aligned 8x8 patches from a larger 40x40 window around each feature point. These descriptors were bias and gain-normalized to ensure consistency. Below is an example of extracted feature descriptors.

Feature Descriptors Example

Step 4: Feature Matching

For feature matching, we identified pairs of features that are similar based on their descriptors. We employed Lowe's ratio test to filter matches by comparing the ratio between the first and second nearest neighbors, as illustrated in Figure 6b of the paper. This approach helps in selecting reliable matches.

Feature Matching Results

Step 5: RANSAC for Homography

Using the matched feature points, we applied 4-point RANSAC to compute a robust homography estimate. This method helps in mitigating the effect of outliers and ensures an accurate transformation matrix for image warping. Below is an example of the homography computation using RANSAC.

RANSAC Homography Computation

The coolest thing I learned from this project was the intricate process of feature matching and how robust algorithms like RANSAC can effectively handle outliers to compute accurate homographies.
